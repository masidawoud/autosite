CMS Approach Options

Three directions — each with a different trade-off between cost, build effort, and client UX

A

Decap CMS + GitHub repos

How it works: Each client gets a GitHub repo containing their site.json, theme.json, sections.json. Decap CMS (open source) sits on top and edits those files via a web UI. On save → GitHub Action clones latest dental-template → Astro build → Cloudflare Pages deploy.


What you get: Form-based editing for structured content (opening hours, team members, services). Markdown editor for WYSIWYG blocks. Drag-and-drop section ordering via list widget. Menu structure via nested list widget.


Pros

  • Free — no platform cost
  • Fits planned architecture exactly
  • Template updates push to all clients via one script
  • Content lives in git — full history

Cons

  • Decap UI is dated and form-heavy
  • WYSIWYG is markdown-only (no visual block editor)
  • GitHub account required per client (or OAuth app setup)
  • Section drag-and-drop is clunky in Decap

Cost: €0/month infrastructure. GitHub free tier. Cloudflare Pages free tier.

B

TinaCMS (self-hosted)

How it works: TinaCMS is a git-backed CMS with visual in-context editing — clients see their actual site in the browser and click to edit text, images, reorder sections. Content still saved to files in a per-client GitHub repo. Same build pipeline as option A.


What you get: Visual editing directly on the rendered page. Rich text editor (not just markdown). Drag-and-drop section ordering. Page builder with predefined blocks. Self-hostable backend (TinaCMS server).


Pros

  • Best client UX — edit in context on real site
  • Git-backed — same architecture direction
  • Rich text is first-class (not just markdown)
  • Self-hostable = no per-client cost
  • Astro integration exists

Cons

  • Requires hosting the TinaCMS server yourself
  • More complex setup per client
  • Younger project, smaller community than Decap
  • Visual editing requires site to be running (preview server)

Cost: Server hosting ~€5–10/month flat (shared across all clients). TinaCMS Cloud: free up to 2 users/site.

C

Sanity (hosted headless CMS)

How it works: Each client gets a Sanity dataset (project). Sanity Studio is the editing UI — a polished React app hosted by Sanity. Content is stored in Sanity's cloud. On publish → webhook → GitHub Action → Astro fetches content from Sanity API → build → Cloudflare Pages deploy.


What you get: Best-in-class rich text (Portable Text — real block editor, not markdown). Native drag-and-drop arrays for sections. Schema-driven — you define exactly what clients can edit. Polished, professional UI that feels like a real product.


Pros

  • Best WYSIWYG and editing experience
  • Most mature, best-supported platform
  • Schema-driven = precise control over what clients can touch
  • No server to host yourself
  • Built-in image handling (CDN, transforms)

Cons

  • €15–20/project/month at scale
  • Content leaves git — no file-based history
  • Astro integration requires rewriting data layer (away from JSON files)
  • Harder to push template updates — data model changes need migration
  • Vendor dependency

Cost: Free for 3 projects. €15/project/month after. 50 clients = €750/month.
