index.astro — section iteration
Instead of importing all 11 components statically, reads sections.json and renders only enabled sections in order. Each section component stays unchanged — just how they're composed changes.
[...slug].astro — dynamic page routing
Reads all files in data/pages/ at build time (getStaticPaths), renders each using the appropriate page template (treatment / info / blank) + the WYSIWYG block if present. This enables unlimited client pages with no template changes needed per page.
Nav.astro — menu from menu.json
Currently hardcoded nav links. Reads menu.json and renders items dynamically, supporting one level of nested dropdowns. Anchors for home sections (#services, #contact), hrefs for pages (/behandelingen/implantaten).
RichText.astro — WYSIWYG renderer
New component that renders TinaCMS rich text (stored as JSON) into HTML. Handles headings, paragraphs, bold/italic, images, bullet lists, simple tables. Used in page templates and optionally in section WYSIWYG blocks.
tina/config.ts — CMS schema
Defines all 5 collections (Business, Theme, Sections, Pages, Menu) with their fields, validation, and UI labels in Dutch. This is the most time-intensive piece — needs to cover every editable field across all sections and page templates.
Existing 11 section components: unchanged — they still receive props, just sourced from the new content model instead of a flat site.json import.
