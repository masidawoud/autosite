Content Model

Five TinaCMS collections — each maps to a file or folder in the client repo

Business Info
Name, address, phone, email, Google reviews score/count/url
site.json → business
Theme
Accent color (color picker), style preset (dropdown: warm-editorial / ocean-depths / tech-innovation)
theme.json
Home Page Sections
Ordered list of sections — drag to reorder, toggle to enable/disable, add from section library.

Each section type has its own fields:
Hero (headline, description, CTA), Quote, Features (6 items), Services (3 cards), Team (members), Reviews, Opening Hours, Insurance, Contact
sections.json (order + enabled)

site.json (section content)
Pages
Each page has: title, slug, template (treatment / info / blank), structured fields from template, + optional WYSIWYG block.

WYSIWYG supports: headings, paragraphs, bold/italic, images, simple tables, bullet lists.
pages/[slug].json

One file per page
Menu
Ordered list of nav items. Each item: label, link (home section anchor or page slug), optional children (one level of nesting = dropdown).
menu.json
What clients cannot touch: Component HTML/CSS, template structure, font definitions, image CDN URLs hardcoded in templates. These stay in dental-template under operator control.
